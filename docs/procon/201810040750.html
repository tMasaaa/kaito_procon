# AtCoder ABC 045 D すぬけ君の塗り絵/Snuke's Coloring

問題

縦HHH, 横WWWの白いマス目のうちNNN個を黒く塗りつぶす。以下のものの個数を各jjjに対し計算する。

  • 各整数j(0≤j≤9)j(0 \leq j \leq 9 )j(0≤j≤9)について、盤に含まれる333行333列の連続するマス目のうち、黒いマスがちょうどjjj個あるもの。
  • 方針
  • まずHHH, WWWが10910^910​9​​なのがやばい。普通に333行333列を全探索しても終わらないことが分かる。
  • 次に、黒いマス目は高々10510^510​5​​個なので、ここに対してアプローチしたいと思う。
  • 二つの方針が思いつく。
    • 座圧する→白の領域が広いので、白の領域を縮めることはできないか？→333行333列のマス目がつぶされてしまうので難しい。
    • 黒のマス目の周辺を探索する(i)(i)(i)
  • 今回は(i)(i)(i)の方針を考える。(一次元で試して思いついたので、二次元で詰まったら次元を落としてみる)

  • 上のように、黒のマスが影響を及ぼすのは周辺555行555列なので、333行333マスの左上に着目して高々9N9N9N個に対して探索すればよいとわかる。
  • ここで333行333列のマス目と左上の点は一対一に対応しているので、mapで左上の点を数えていくと、333行333列のマス目の中に黒いマス目がいくつあるのかをカウントできる。(各黒マスを一回数えるごとに、map[座標]の値が一つずつ増えていく)
  • 以上をコードにする。計算量は9N9N9Nでmapを使っているのでO(N)O(N)O(N)か？
#include <bits/stdc++.h>
using namespace std;

int64_t a,b,H,W;
int64_t cnt[10];
map<pair<int64_t, int64_t>, int64_t> mp;
int N,i,j,k;
int64_t sm;
int main(){
  std::ios::sync_with_stdio(false);
  std::cin.tie(0);

  // ifstream in("input.txt");
  // cin.rdbuf(in.rdbuf());

  cin>>H>>W>>N;
  for(i=0;i<N;i++){
    cin>>a>>b;
    a--;b--;
    for(j=-2;j<=0;j++){
      for(k=-2;k<=0;k++){
        if(a+j<0 || a+j+2>H-1 || b+k<0 || b+k+2>W-1)continue;
        mp[make_pair(a+j, b+k)]++;
      }
    }
  }
  sm = (H - 2) * (W - 2);
  for(auto itr=mp.begin(); itr!=mp.end();++itr){
    for(i=1;i<=9;i++){
      if((itr->second)==i)cnt[i]++;
    }
  }
  for(i=1;i<=9;i++){
    sm-=cnt[i];
  }
  cnt[0]=sm;
  for(i=0;i<=9;i++)cout << cnt[i] << endl;
  return 0;
}

まとめ

  • 典型: 正方形グリッドを見るときは左上の点と一対一対応させる。
  • 1-indexedと0-indexedに気を付ける
  • 10910^910​9​​が入るときはmapに持たせることを考える。(配列には持てないため)
