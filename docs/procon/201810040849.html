# AtCoder ABC 047 D 高橋君と見えざる手 / An Invisible Hand

問題

買うリンゴ、売るリンゴ合わせてTTT個以下という制約のなかで最大の利益を出すよう旅をする高橋君を、青木君が邪魔する(利益が少なくとも111円下がるようにする)

  • 方針
  • まず、町iiiでリンゴを売るときを考える。このとき、利益を最大にするよう動いているため町111, 222, ..., i−1i-1i−1の中で一番安い場所で買ったリンゴを売ることになる。したがって、一個リンゴを買い、一個リンゴを売るとしたときの利益をBiB_iB​i​​と置くと、Bi=Ai−min{A1,...,Ai−1}B_i = A_i - \min\{ A_1, ..., A_{i-1} \}B​i​​=A​i​​−min{A​1​​,...,A​i−1​​}
  • ここで、制約TTTが小さいときを考えると、BiB_iB​i​​が最大となる町iiiで必ずリンゴを売ることが分かる(TTTが大きいときは大きい方からBiB_iB​i​​をとっていくようになるので)
  • したがって、BiB_iB​i​​が最大になる町iiiに対し青木君がAiA_iA​i​​を1下げる操作を行えばよい。
  • ここで、最大となるBiB_iB​i​​が複数あった場合、BiB_iB​i​​ひとつに対し操作を行っただけでは、高橋君が他のBiB_iB​i​​を最大にするiiiを選べば利益を下げることができないので、BiB_iB​i​​を最大にするiiiすべてに対し操作を行う必要がある。(最大値の複数候補)
  • 次に、これらAiA_iA​i​​が重複することはないか考える。Bi=Bi′B_i = B_{i'}B​i​​=B​i​′​​​​ならばAiA_iA​i​​が相異なるという条件からmin{A1,...,Ai−1}\min\{ A_1, ..., A_{i-1} \}min{A​1​​,...,A​i−1​​}は一意に定まり、これをCiC_iC​i​​と置くとAi−Ci=Ai′−Ci′A_i - C_i = A_{i'} - C_{i'}A​i​​−C​i​​=A​i​′​​​​−C​i​′​​​​となる。ここで、AiA_iA​i​​が相異なるという条件からCiC_iC​i​​も相異なるので、重複はない(ダブルカウントのチェック)
  • 以上をコードにする。計算量はO(N)O(N)O(N)
#include <bits/stdc++.h>
using namespace std;

#define LINF 1000000000000000001 // 1e18+1

map<int,int64_t> mp;
int i,j,k,N;
int64_t T,a;
int main(){
  std::ios::sync_with_stdio(false);
  std::cin.tie(0);

  // ifstream in("input.txt");
  // cin.rdbuf(in.rdbuf());

  cin >> N >> T;
  int64_t mn=LINF;
  for(i = 0;i < N;i++){
    cin >> a;
    mn = min(mn, a);
    mp[a - mn]++;
  }
  int64_t index = 0,ans = 0;
  for(auto itr=mp.begin(); itr!=mp.end();++itr) {
    j = (itr->first);
    if(index < j){
      index = j;
      ans = (itr->second);
    }
  }
  cout << ans << endl;
  return 0;
}
