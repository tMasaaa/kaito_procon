# AtCoder ABC 112 C Pyramid

  • 全探索を思いつくまでと、h[i]=0h[i]=0h[i]=0の処理が厳しかった。

問題

座標(X,Y)(X, Y)(X,Y)の高さをmax(H−∣X−Cx∣−∣Y−Cy∣,0)\max (H - |X - Cx| - | Y - Cy|, 0)max(H−∣X−Cx∣−∣Y−Cy∣,0)で定める。x,y,hx, y, hx,y,hの組がNNN個与えられるので、ピラミッドの中心(Cx,Cy)(Cx, Cy)(Cx,Cy)と高さHHHを求める。

  • 方針
  • 最初は数学的に解こうとしてしまうが、求めるものに注目する。0≤Cx≤1000 \leq Cx \leq 1000≤Cx≤100, 0≤Cy≤1000 \leq Cy \leq 1000≤Cy≤100を見ると、中心部について全探索が可能であるとわかる。(すべての基本は全探索)
  • h[i]≠0h[i] \neq 0h[i]≠0のとき、この点はピラミッドの上にあるので、HHHについての式を変形して、H=∣x[i]−cx∣+∣y[i]−cy∣+h[i]H=|x[i]-cx| + |y[i]-cy| + h[i]H=∣x[i]−cx∣+∣y[i]−cy∣+h[i]としてHHHを定められる。(ここで、h[i]h[i]h[i]は1以上なのでHHHも1以上)
  • このHHHがある点x1,y1,h1x1, y1, h1x1,y1,h1とx2,y2,h2x2,y2,h2x2,y2,h2で食い違うときはCx,CyCx, CyCx,Cyは中心でないとわかるのではじく。
  • さらに、h[i]=0h[i] = 0h[i]=0の場合を考える。ピラミッドのふちにある点なのか、それともピラミッドに関係ないただの面上の点なのか判定する。
  • 判定後にHHHが食い違うことがないか確かめる
  • 以上の判定を抜けたものを出力する。
  • これらをコードにする。計算量はO(100×100×N)O(100 \times 100 \times N)O(100×100×N)
#include <bits/stdc++.h>
using namespace std;

int N,i;
int64_t h[110],x[110],y[110];
int cx,cy;
int64_t memo=-1,H;
int main(){
  std::ios::sync_with_stdio(false);
  std::cin.tie(0);

  // ifstream in("input.txt");
  // cin.rdbuf(in.rdbuf());

  cin >> N;
  for(i = 0;i < N;i++){
    cin >> x[i] >> y[i] >> h[i];
  }

  for(cx = 0;cx <= 100;cx++){
    for(cy = 0;cy <= 100;cy++){
      memo = -1;
      for(i = 0;i < N;i++){
        if(h[i] == 0)continue;
        H = abs(x[i]-cx) + abs(y[i]-cy) + h[i];// 1以上
        if(memo == -1){
          memo = H;
        }else{
          if(memo != H){
            goto E;
          }
        }
      }
      memo = H;
      for(i = 0;i < N;i++){
        if(h[i] != 0)continue;
        if(H - abs(x[i]-cx) - abs(y[i]-cy) < 0)continue;
        H = abs(x[i]-cx) + abs(y[i]-cy) + h[i];
        if(memo != H){
          goto E;
        }
      }
      cout << cx <<" "<< cy <<" "<< H << endl;
      return 0;
      E:;
    }
  }
  return 0;
}
